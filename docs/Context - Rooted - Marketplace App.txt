## 1. PRODUCT OVERVIEW & CORE VISION

Rooted is a mobile-first, two-sided local commerce and event marketplace designed to bridge the gap between digital pre-ordering and physical, live community events (markets, conventions, pop-ups, festivals). 

### Major Problems Solved
* **The Discovery Problem:** Shoppers lack a centralized hub to know which vendors exist, where they are setting up, and what events are active nearby.
* **The Sales Problem:** Vendors miss critical pre-event revenue because shoppers cannot browse stock before arriving physically at a venue.
* **The Communication Problem:** No single platform exists for local merchants to broadcast real-time stock alerts, location shifts, and event schedules directly to interested locals.

Rooted serves as a **local discovery machine**, an **organic vendor promotional engine**, and an **agile pre-order management tool** rolled into a single system.

## 3. CORE ARCHITECTURE & TARGET TECH STACK

* **Frontend Framework:** Expo SDK 54 (React Native) initialized via the TypeScript native `tabs` layout template.
* **Styling Engine:** TailWind CSS via NativeWind for fluid utilities.
* **Backend Platform:** Supabase (PostgreSQL Database, GoTrue Native Auth, Storage Buckets).
* **V1 Transaction Model:** **Model A: Reserve and Pay at Pickup.** Shoppers reserve items in-app, vendors verify/approve, and payment happens offline/in-person at the booth. This avoids premature checkout complexity, payment gateways, processing fees, and dispute infrastructure at launch.
* **Strict Execution Directive:** Mobile-first architecture. **No web-exclusive browser APIs (`window`, `document`, etc.)** are permitted within active app routes unless wrapped inside an explicit platform check (`Platform.OS === 'web'`).

## 4. MVP CORE FUNCTIONAL MODULES

### A. The Shopper Experience
* **Discovery Dashboard:** A dynamic home screen emphasizing upcoming events near the user, featured local vendors, active map previews, and a live product catalog feed.
* **Multi-View Event Discovery:** Event browsing via searchable list layouts, chronological calendars, or location-focused map pins marking active venues.
* **Storefront Profiles:** Deep-dive look into individual vendor booths displaying custom bios, linked social handles, active schedules, and product listings.
* **Reserve-for-Pickup Checkout:** A zero-friction wizard allowing shoppers to select items, allocate them to an upcoming event pickup location, and monitor real-time fulfillment states.
* **Subscriber Feed:** A curated community feed delivering updates, restocks, and promotional alerts purely from vendors the shopper has flagged as a "Favorite."
* **Contextual Messaging:** Deeply sandboxed chat routing allowing communication between shoppers and vendors *only* when linked directly to active order receipts or specific inquiries.

### B. The Vendor Experience
* **Operational Dashboard:** High-level metrics window tracking raw sales volumes, pending order tallies, low-inventory triggers, and quick-action shortcuts for swift updates.
* **Storefront Customization Engine:** A lightweight profile manager giving vendors codeless authority over profile logos, banner layouts, color styling, and bio write-ups.
* **Dual-Channel Inventory Controls:** A critical business safeguard. Vendors can partition total stock into two distinct channels: **Pre-Sale Digital Stock** (buyable via the app) and **In-Person Booth Stock** (protected for physical walk-ups). This completely blocks digital sales from depleting physical counter tables.
* **Order Lifecycle Manager:** Step-by-step state toggles supporting order acceptance, rejections, packaging phases, ready-for-pickup alerts, and fulfillment tracking.
* **Broadcast Publisher:** Content tool enabling vendors to fire rich media posts directly into the shopper live feeds to prompt real-time local engagement.
* **Analytics & CSV Exporters:** Data layout summarizing revenue velocity, high-performing product items, and event-by-event performance. Supports clean CSV tabular data extractions.
* **Manual Offline Tracking:** An internal logging menu allowing vendors to manually type in cash/card sales processed at physical events, keeping the in-app analytics engine completely accurate to their real-world numbers.

## 6. BUSINESS RULES & PLATFORM LOGIC

* **Account Bound:** A profile selects an identity context during configuration. Cross-role conversions (e.g., a Shopper applying to list products as an authorized Vendor) require manual Admin verification flags.
* **Contextual Storefront Constraints:** Vendor products must be explicitly mapped to an existing `Event` instance via `ProductEventAvailability` profiles to become available for digital reservation. Items not mapped default to a display-only state.
* **Pre-Sale Allocation Logic:** Digital checkouts must block execution if requested quantities cross over the isolated `available_quantity_presale` limits.
* **Fulfillment Auto-Deductions:** Transitioning an order row to `fulfilled` must trigger an automatic negative entry inside `InventoryTransactions` to properly balance database snapshots.
* **Feed Ranking Controls:** Feed content sorting must weigh localized geographic ranges, chronological recency, and user interest taxonomies above global sorting vectors.
